## 完整题库（43 题）

### 一、Agent 项目相关

1. **为什么做 Agent 项目？**
   - 动机：解决什么问题？为什么用 Agent 而不是普通 LLM？
   - 亮点：项目有什么独特之处？

2. **了解过市面上有哪些智能体 Agent 吗？**
   - 主流框架：LangChain、LangGraph、CrewAI、AutoGen、Coze、Dify
   - 产品：Claude Code、Copilot、Devon、Devin

3. **讲下 Agent 项目（整体架构 & 执行流程）**
   - 架构图：用户输入 → 意图识别 → 工具调用 → 结果整合 → 输出
   - 流程：ReAct 循环（思考→行动→观察→重复）

4. **Agent 项目开发的框架**
   - LangGraph（状态图）、LangChain（链式）、CrewAI（多Agent编排）
   - 选型理由：为什么选这个框架？

5. **Agent 项目背景**
   - 项目起源、业务场景、解决的用户痛点
   - 技术选型考量

6. **了解其他的 Agent 范式吗？**
   - ReAct、Plan-and-Execute、Reflexion、AutoGPT、Task Decomposition
   - 各种范式的优缺点对比

7. **Agent 推理模式**
   - ReAct（推理+行动循环）
   - Plan-and-Solve（先规划再执行）
   - Reflection（自我反思纠错）
   - 推理模式的差异化设计

8. **多 Agent 执行策略的智能选择和切换机制设计**
   - 根据任务复杂度动态选择单Agent/多Agent
   - 路由策略：分类器 → 选择Agent
   - 切换条件：超时/失败/置信度过低

9. **举例复杂任务下执行流程**
   - 比如"帮我写一份AI行业报告"：搜索→整理→分析→生成→检查
   - 每一步的输入输出、分支处理、异常情况

10. **Agent Skills**
    - 可插拔的能力模块（搜索、计算、代码执行等）
    - Skill 的注册、发现、调用机制
    - 与 MCP 的关系和区别

11. **复杂任务执行准确率提升的评估方法**
    - 任务分解粒度
    - 中间结果验证
    - 回滚重试机制
    - 评估指标：成功率、完整度、耗时

12. **跨模块错误追踪的 Agent 知识库构建方案**
    - 错误类型分类→存档→检索→复用
    - 每次出错记录上下文、错误信息、修复方案
    - 新错误先在知识库中检索相似案例

13. **演示 Agent 项目实现细节**
    - 核心代码片段解读
    - 关键模块的输入输出
    - 配置文件和参数说明

14. **项目中 AI 贡献的代码占比**
    - 如实回答，但要强调：哪些部分是你设计的、调试的、决策的
    - 重点讲"人的价值"：架构设计、方案选型、bug修复、效果优化

### 二、RAG 相关

15. **RAG 系统全流程**
    - 索引流程：文档→切分→向量化→存储
    - 查询流程：问题→向量化→检索→重排序→生成
    - 重点环节：分块策略、Embedding模型选择、检索融合

16. **RAG 检索优化策略**
    - 混合检索（BM25 + 向量检索 + RRF融合）
    - 重排序（Reranker 精排）
    - 多阶段召回（粗召回→精排）
    - Query 扩展（多Query、HyDE）

17. **RAG 评估方案**
    - RAGAS 框架：忠实度、相关度、上下文精度、召回率
    - 人工评估 + 自动化评估结合
    - 评估数据集构建

18. **多阶段召回策略优化**
    - 第一层：关键词+向量粗召回（Top-50）
    - 第二层：Reranker精排（Top-5）
    - 第三层：去重+时效性过滤
    - 各层的召回率/准确率平衡

19. **RAG 动态知识更新**
    - 增量更新：只处理新增/变更的文档
    - 定时重建：定期全量重建索引
    - 版本管理：旧版本数据保留，支持回滚

20. **企业内部知识库 RAG 的动态持续更新方案**
    - 监控文档源变化
    - 增量 Embedding
    - 索引版本控制
    - 平滑切换（蓝绿部署）

### 三、MCP / Function Calling / Tool 调用

21. **MCP（Model Context Protocol）核心内容**
    - 定义：AI世界的"USB接口"——标准化工具调用协议
    - 架构：MCP Client ↔ MCP Server ↔ 工具/数据源
    - 传输层：stdio（本地）、SSE（远程）

22. **MCP 和 Function Calling 的区别**
    - Function Calling：每个框架自己定义格式，不通用
    - MCP：统一协议，一次接入到处用
    - MCP 的 Server/Client/Tool 三层架构

23. **MCP 和 Skill 的区别**
    - MCP：协议层标准，定义通信方式
    - Skill：能力层抽象，定义"能做什么"
    - Skill 可以基于 MCP 实现

24. **A2A 协议**
    - Agent-to-Agent，Agent之间的通信协议
    - 与 MCP 的区别：MCP 是 Agent↔工具，A2A 是 Agent↔Agent
    - 适用场景：多Agent协作

25. **A2A 与 MCP 区别**
    | 维度 | MCP | A2A |
    |------|-----|-----|
    | 通信方向 | Agent ↔ 工具 | Agent ↔ Agent |
    | 定位 | 工具标准化 | Agent 协作标准化 |
    | 核心能力 | 工具发现/调用 | 任务协商/结果共享 |

26. **特定推理模型不支持 MCP 的技术原因**
    - 模型不支持 Tool Calling / 不支持结构化输出
    - 解决：通过中间层转换（如封装一个 Proxy Agent）

27. **MCP 通信方式**
    - stdio（本地进程通信）
    - SSE（Server-Sent Events，远程通信）
    - Streamable HTTP（流式HTTP）

28. **Tool 调用机制**
    - 定义 Tool Schema（参数+返回）
    - LLM 生成 Tool Call 请求
    - 执行 Tool 并返回结果
    - 结果融入对话上下文

### 四、大模型基础

29. **介绍一些 AI 大模型**
    - GPT-4o（OpenAI 多模态旗舰）
    - Claude Opus 4.8（Anthropic，长上下文强项）
    - DeepSeek V4 Flash（国产开源，1M上下文）
    - Qwen 3.7 Plus（阿里，视觉能力强）
    - Gemini 2.5 Pro（谷歌，多模态原生）

30. **LLM 产生幻觉的原因及解决方案**
    - **原因**：模型本质是概率生成、训练数据有偏差、缺乏真实世界知识
    - **方案**：RAG（检索增强）、Prompt约束、温度调低、事实核查Agent

31. **推理模式的差异化设计**
    - 快速推理（低温度、短链）→ 日常问答
    - 深度推理（CoT、高温度）→ 复杂问题
    - 混合模式：先快推，置信度低时切深推

32. **模型预热机制**
    - 模型首次加载时慢（冷启动）
    - 预热：提前发送请求让模型加载到缓存
    - 适用于 Serverless 部署场景

33. **NL2SQL 场景下的 SQL 安全防护**
    - 限制权限（只读、限定表）
    - SQL 注入检测
    - 结果行数限制
    - 敏感字段脱敏

### 五、Prompt 工程

34. **如何写好 Prompt？**
    - 明确角色 + 任务 + 输出格式
    - 提供示例（Few-shot）
    - 分步骤引导（Chain-of-Thought）
    - 加约束条件（不该做什么）

35. **Prompt 工程的实践经验**
    - 结构化 Prompt（角色/任务/要求/输出格式）
    - 迭代优化（A/B测试）
    - 版本管理
    - 量化评估（成功率、格式合规率）

36. **Prompt 设计示例**
    ```
    你是一个{角色}。
    任务：{具体任务}
    要求：
    1. {约束1}
    2. {约束2}
    输出格式：{JSON/文本/列表}
    示例：{optional}
    ```

### 六、工程实践

37. **多轮对话的实现方案**
    - 上下文管理：保留最近N轮 + 早期摘要压缩
    - 记忆系统：短期（窗口）+ 长期（数据库/向量库）
    - 状态跟踪：当前话题、已完成/未完成意图

38. **SSE 的局限性**
    - 单向通信（只能服务端→客户端）
    - 连接数限制（浏览器并发限制）
    - 断线重连复杂
    - 适合实时推送，不适合双向交互

39. **长文本生成的技术方案**
    - 分块生成（大纲→分段→组合）
    - 滑动窗口（保持中间段的上下文）
    - 递归摘要（先生成摘要再扩展）
    - 流式输出（用户边看边等）

40. **基于代码构建知识库的 Agent 设计**
    - 代码解析（AST提取结构）
    - 知识图谱（函数/类/文件之间的关系）
    - 自然语言检索（"支付流程在哪"）
    - 增量更新（只处理变更文件）

41. **AI 辅助 IDE 开发工具**
    - Claude Code、Cursor、Copilot、Codex
    - 各有何特点、适用场景
    - 你实际使用的经验和感受

42. **AI 辅助开发的实践经验**
    - 用 AI 写代码的流程：需求→Prompt→生成→审查→修改
    - 常见问题及应对：幻觉、遗漏、安全隐患
    - 人机协作的最佳方式：AI生成+人工审查

43. **简历关键词提取的技术实现**
    - 规则提取（正则匹配技能词）
    - LLM 提取（让模型分析简历）
    - 混合方案：规则粗筛 + LLM精提

## 面试回答原则

1. **先讲整体，再讲细节** — 不要一上来就扎进代码
2. **每个问题都要落脚到"你做了什么"** — 不只是背概念
3. **讲踩过的坑比讲完美的方案更加分** — 真实即力量
4. **不知道就说不知道** — 不要编，可以说"我没接触过，但我推测可能是..."
5. **控制时长** — 一个问题回答控制在 2-3 分钟
